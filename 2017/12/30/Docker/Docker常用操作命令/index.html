Docker常用操作命令

  • 2017-12-30

Docker 三大组件

仓库：存放各种各样已经打包好的Docker应用
镜像：按照Docker的规则制作的应用。（像Windows 安装软件包）
容器：；利用镜像创建容器：启动预先定义好的进程与用户交互，对提 供服务。一个镜像可以创建多个名字不同的容器

基础命令

启动 查询

容器启动：docker run –it/-d –p/P –name imageID/name

容器启动/重启/停止: docker start/restart/stop 容器id

容器进入：docker attach containerID 这个进入退出的话，容器会停掉
docker exec –it containerID /bin/bash

容器查询：docker inspect containerID
docker top containerID
docker diff containerID

列出本地镜像

  • -q 只显示镜像ID

删除容器

  • -f :通过SIGKILL信号强制删除一个运行中的容器
  • -v :-v 删除与容器关联的卷

删除镜像

  • -f 强制删除

更新镜像

  • latest 标记最新版本
    -

创建并运行容器

  • 以上wordpress由apache 和 mariadb 组成
  • 多容器是由link进行连接由数据。条件 –link db:mysql必须必wordpress先创建并运行
  • 删除或停止也需要按顺序 先 wordpress 在 mysql

-name 指定容器名字
-e 指定环境变量，容器中可以使用该环境变量
-d 指定容器运行于前台还是后台
-p 指定容器暴露的端口
–link 指定容器间的关联
-i 打开STDIN，用于控制台交互
-t 分配tty设备，该可以支持终端登录

查找镜像 版本

获取容器/镜像的基本信息

  • 基本信息：存储位置，配置参数，网络设置等

-f :指定返回值的模板文件。
-s :显示总的文件大小。

运行的容器中执行命令

-d :分离模式: 在后台运行
-i :即使没有附加也保持STDIN 打开
-t :分配一个伪终端

从容器创建一个新的镜像

-a :提交的镜像作者；
-c :使用Dockerfile指令来创建镜像；
-m :提交时的说明文字；
-p :在commit时，将容器暂停。
